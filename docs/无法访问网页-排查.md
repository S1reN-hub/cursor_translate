# 网页无法访问 — 排查步骤

## 1. 确认开发服务器真的在运行

执行 `npm run dev`（或 `& "C:\Program Files\nodejs\npm.cmd" run dev`）后，终端里**必须**出现类似：

```
▲ Next.js 14.x.x
- Local:        http://localhost:3000
✔ Ready in 2.3s
```

- **若没有这段**：说明服务没起来。看终端里是否有**红色报错**，把完整报错复制下来再排查。
- **若有这段**：说明服务已启动，继续下面步骤。

---

## 2. 确认访问的地址

在浏览器地址栏输入（任选其一，不要漏掉 `http://`）：

- **http://localhost:3000**
- 或 **http://127.0.0.1:3000**

不要用 `https://`，本地开发一般是 `http`。

---

## 3. 端口被占用（常见）

若终端里出现 **Port 3000 is in use** 或 **EADDRINUSE**：

- 说明 3000 端口已被其他程序占用，Next 可能没真正监听，或改用了别的端口。
- **处理方式一**：关掉占用 3000 端口的程序，再重新执行 `npm run dev`。
- **处理方式二**：换一个端口启动，在项目根目录执行：
  ```bash
  npm run dev -- -p 3001
  ```
  然后访问 **http://localhost:3001**。

---

## 4. 用 CMD 重新启动一次（避免 PowerShell 干扰）

1. 在 Cursor 里 **终端 → 新建终端**，确认终端类型是 **Command Prompt (cmd)** 而不是 PowerShell。
2. 执行：
   ```bash
   cd D:\project\cursor_teanslate
   npm install
   npm run dev
   ```
3. 等出现 `Ready` 和 `Local: http://localhost:3000` 后，再在浏览器打开 http://localhost:3000 。

---

## 5. 防火墙 / 安全软件

若以上都正常，仍打不开页面：

- 临时关闭 Windows 防火墙或第三方安全软件，再试一次。
- 或在该软件里为 **Node.js** 或 **localhost** 放行。

---

## 6. 把终端输出发给别人排查

若仍无法访问，请：

1. 在项目根目录执行 `npm run dev`。
2. 等待 10～20 秒。
3. 把终端里**从执行命令到结束的整段文字**（包括报错）复制下来，发给能帮你的人或贴到问题里。

这样可以看到是编译报错、端口占用，还是其他原因导致无法访问。
